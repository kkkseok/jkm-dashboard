# 마이너스 매출이익률 기능 — 요구사항

> 작성: 2026-05-22 / 출처: 사용자 대화 + `memory/project_minus_logic.md`

## 사용자 시나리오

판매채널 운영 담당자가 **매일 매출 마감 후** 손실 발생 품목을 확인한다. 두 개의 엑셀(`sales_status_basic.xlsx`, `revenue_profit_product.xlsx`)을 업로드하면 시스템이 매핑·계산 후 결과를 테이블로 보여준다. 상품코드별 "추가후정산금"은 시스템 내부 테이블(`cal_amount`)을 통해 별도 관리한다.

## 입력

매일 업로드 (사용자가 직접):
1. **`sales_status_basic.xlsx`** — 기본 데이터. 병합 헤더 2행. 약 33 컬럼.
2. **`revenue_profit_product.xlsx`** — 상품코드·상품명 참조용. 병합 헤더 2행.

별도 관리(웹 CRUD):
3. **`cal_amount`** — 상품코드 → 추가후정산금 룩업. 초기 1회 `docs/common/cal_amount.xlsx` import 후 웹에서 추가/수정/삭제.

## 가공 로직

`memory/project_minus_logic.md` 및 `.claude/skills/profit-calc/skill.md` 참조.

핵심 매핑:
- `sales_status_basic.AE`(온라인주문번호) ↔ `revenue_profit_product.E`(주문번호)
- → `Y`(상품코드), `AG`(상품명) 가져오기
- → 상품코드로 `cal_amount` 룩업 → 추가후정산금

5개 신규 계산 컬럼:
- `수수료 = 1 − L/K`
- `후정산금 = K × (수수료/2)`
- `추가후정산금 = cal_amount 룩업 (없으면 0)`
- `총마진액 = R + 후정산금 + 추가후정산금`
- `총마진율 = 총마진액 / L`

## 출력

화면 2개:
- **분석 페이지** — 두 파일 업로드 → 매핑·계산 → 결과 테이블 + 요약 KPI
- **추가후정산금 관리 페이지** — `cal_amount` CRUD

표시 필수 컬럼 (분석 페이지 결과 테이블):
- 매출일, 온라인주문번호, 상품코드, 상품명
- 매출액(K), 공급가(L), 이익액(R) — 기존
- 수수료, 후정산금, 추가후정산금, 총마진액, 총마진율 — 신규

추가 표시 컬럼은 `uiux-designer`가 사용성을 고려해 결정.

## 보류 항목

- **"마이너스 품목" 판정 기준 컬럼** — 총마진율 < 0 / 총마진액 < 0 / R < 0 중 미정. UI는 필터 자리만 미리 만들고, 기본은 "전체 표시"로 둠. 사용자가 추후 확정 시 즉시 반영.
- **음수 임계값** — 0 미만 모두 vs 특정 % 이하 등. 동일하게 보류.
