# 대시보드 UX 패턴

`jkm-dashboard`처럼 **데이터 밀도가 높은 사내 도구**의 화면 설계 원칙. 본 문서를 참조하여 `uiux-designer`는 명세를 작성하고, `next-builder`는 구현 시 일관성을 유지한다.

## 1. 정보 아키텍처 (IA)

- **기능 단위로 좌측 사이드바 또는 상단 탭.** `jkm-dashboard`는 기능이 점진적으로 늘어남(minus → soldout → group). 사이드바 권장.
- **한 화면 = 한 의도.** "마이너스 품목 확인" 화면에 품절 관리까지 끼우지 않는다.
- **CRUD 관리 화면은 본 화면과 분리.** `cal_amount` 관리는 별도 페이지/탭. 분석 화면이 CRUD로 어지러워지지 않게.
- **breadcrumb 또는 페이지 제목으로 위치 명시.** 사용자는 점프해서 들어오는 경우 많다.

## 2. 레이아웃 골격

대시보드 분석 화면의 표준:
```
+----------------------------------------------------+
| Page Title              [업로드] [다운로드] [⚙]    |
+----------------------------------------------------+
| 요약 카드: KPI 3~5개 (총 N건, 마이너스 N건, …)    |
+----------------------------------------------------+
| 필터: 기간 [드롭다운] 채널 [select] 검색 [input]  |
+----------------------------------------------------+
| 데이터 테이블 (정렬 가능 헤더, 페이지네이션)        |
|  | 컬럼A ▲ | 컬럼B | … | 액션 |                   |
+----------------------------------------------------+
```

CRUD 관리 화면:
```
+----------------------------------------------------+
| Page Title                          [+ 추가]       |
+----------------------------------------------------+
| 검색 [input]                                       |
+----------------------------------------------------+
| 테이블 (행 클릭 → 수정 dialog)                      |
+----------------------------------------------------+
```

## 3. 데이터 밀도

- **금액·수량 우측 정렬.** 천 단위 구분자(`Intl.NumberFormat('ko-KR')`).
- **비율은 % 표기 + 소수 1자리.** 음수는 빨강(`text-red-600 dark:text-red-500`).
- **행 높이는 보통(`h-10`).** 컴팩트 모드는 토글로 제공해도 OK.
- **컬럼이 많으면 가로 스크롤.** 우선순위 낮은 컬럼은 기본 숨김 + 토글.

## 4. 상태별 UI (필수 4종)

대시보드 어떤 영역이든 아래 4개 상태를 가진다. 빠뜨리면 사용성 무너짐.

| 상태 | UI 패턴 |
|------|---------|
| **빈** | 일러스트 또는 아이콘 + "업로드해주세요" 같은 액션 안내 + 1차 CTA 버튼 |
| **로딩** | shadcn `Skeleton`으로 테이블/카드 골격 표시. 빙글이 아니라 윤곽 |
| **에러** | shadcn `Alert variant="destructive"`. 사용자가 할 수 있는 다음 행동을 명시 |
| **성공/데이터 있음** | 본 콘텐츠 |

## 5. 업로드 UX

- **드래그앤드롭 영역 + 클릭 업로드** 둘 다 제공
- **업로드 중 progress** (파싱 시간이 오래 걸리는 큰 파일 대응)
- **파일 검증 에러는 inline.** 어느 행/시트에서 무엇이 잘못됐는지
- **이전 업로드 결과 보존.** 새 파일로 덮어쓰기 전 사용자 확인
- 여러 파일이 필요한 경우(현재 minus 기능: 2개 파일) **단계 표시기 또는 체크리스트**

## 6. 필터/검색

- **filter chip 패턴.** 적용된 필터는 chip으로 보여주고 X로 해제 가능
- **검색은 즉시 반영(debounce 200~300ms).**
- **자주 쓰는 필터 조합은 저장 가능하면 좋음(향후 확장).**

## 7. 폼 (CRUD)

- **shadcn `Dialog` + `Form` + `react-hook-form` + `zod`.**
- **필수 필드 명시 (*).**
- **인라인 validation 에러.** 폼 제출 후가 아니라 blur 시점에.
- **submit 중 버튼 disabled + 로딩 표시.**
- **성공 시 `Toast` 알림 + Dialog 자동 닫기.**

## 8. 접근성 (체크리스트)

- [ ] 모든 인터랙티브 요소가 키보드로 도달 가능 (`Tab`, `Shift+Tab`)
- [ ] `Esc`로 dialog/dropdown 닫힘
- [ ] `Enter`로 폼 submit
- [ ] 아이콘 버튼에 `aria-label`
- [ ] 텍스트 대비 4.5:1 이상 (Tailwind 기본 팔레트는 대부분 OK, 회색조 주의)
- [ ] 폼 input과 label `htmlFor` 연결
- [ ] 에러 메시지는 `aria-describedby`로 연결

## 9. 반응형

- **데스크탑 우선 설계.** 사내 분석 도구 특성상 모바일은 보조.
- **테이블은 모바일에서 가로 스크롤 허용.** 카드 변환은 데이터 밀도 손실이 커서 권장 안 함.
- **사이드바는 모바일에서 `Sheet`(슬라이드)로 전환.**
- **breakpoint 기준:** `md` (768px)에서 사이드바 모드 전환, `lg` (1024px)에서 풀 레이아웃.

## 10. 디자인 토큰 (초기 기본값)

- **컬러:** shadcn 기본(slate) + accent로 `blue-600`. 마이너스 강조는 `red-600`.
- **타이포:** `font-sans` (Pretendard 또는 시스템 폰트). 숫자는 `tabular-nums`로 자릿수 정렬.
- **간격:** Tailwind 4의 배수(`gap-2`, `gap-4`, `gap-6`)만 사용. 임의 값 금지.
- **다크모드:** shadcn 기본 지원. 초기 라이트 모드만 활성화, 다크는 향후.

## 11. 명세 작성 시 체크리스트 (uiux-designer 산출물)

명세 한 건에 다음이 모두 들어 있어야 한다:

- [ ] 페이지 목적 (1~2줄)
- [ ] IA 위치 (사이드바 어느 항목)
- [ ] ASCII 와이어프레임
- [ ] 컴포넌트 목록 (shadcn 명) + 역할
- [ ] 필드 → UI 컬럼 매핑 표
- [ ] 빈/로딩/에러/성공 4개 상태 명세
- [ ] 인터랙션 시나리오 1~3개 (사용자 클릭 흐름)
- [ ] 키보드/접근성 메모
- [ ] 반응형 동작 (모바일/태블릿)

이 체크리스트가 통과되지 않은 명세는 `next-builder`에 넘기지 않는다.
